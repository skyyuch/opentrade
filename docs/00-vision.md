# OpenTrade 願景

> **這份文件描述「我們在做什麼、為何做、為誰做」。**
> 它是項目的北極星，**不會頻繁更新**。當任何決策不確定時，回到這裡找答案。

---

## 一、項目使命

> **用區塊鏈讓金融服務評論回歸公正：平台無權刪除，無權竄改，無人能用錢洗白。**

OpenTrade 要打造一個讓投資者**敢相信評論、找得到真相**的平台。

---

## 二、我們解決的問題

### 問題 1：Web 2.0 評分平台的「中心化勒索」

以外匯天眼（WikiFX）為例，業界普遍存在以下問題：

- **付費刪負評**：商家付費後，平台「協助處理」負面留言
- **付費買排名**：廣告位影響搜尋結果與評分顯示
- **無證查證**：用戶投訴未經核實就被翻譯成多國語言公開，目的是逼商家付費「處理」
- **球員兼裁判**：平台同時是評分制定者、廣告收益方、爭議仲裁方

### 問題 2：財經 KOL（財演）「贏了高調、輸了刪文」

香港的「財演」與技術指標賣家最常見的行銷手法：

- 公開喊單「打中」就大肆宣傳，截圖留底
- 喊單失敗就刪除原貼文、重發新貼文
- 銷售技術指標時提供精挑細選的歷史回測，不揭露失敗案例

投資者無法取得**完整、不可篡改**的歷史業績紀錄。

### 問題 3：監管的盲區

香港證監會（SFC）監管的是合規與牌照，但**不管**：

- App / 平台的使用者體驗（系統穩定性、出金速度、客服品質）
- KOL 喊單的真實勝率
- 暗藏條款（隱性手續費、滑點、停損機制）

## 三、目標市場

### Phase 1（MVP，2026 Q3-Q4）

聚焦香港，四類角色：

1. **持牌證券商**（SFC 第 1、2 類牌照）
   - 補齊 SFC 不管的「使用體驗」評論
   - 例：富途、輝立、耀才、致富等

2. **財經 KOL / 財演**
   - 鏈上認證喊單歷史，建立可驗證的勝率紀錄
   - 不接受認證者，平台顯示「未經鏈上驗證」紅標

3. **技術指標 / 訊號賣家**
   - 策略訊號智能合約化，永久公開歷史表現

4. **金商 / 貴金屬交易商**（香港黃金交易所 HKGX 行員）
   - 僅限本地（香港）金商；權威名冊來源為香港黃金交易所（HKGX，2025-01-01 起接替前身金銀業貿易場 CGSE）
   - 與證券商同樣補齊「使用體驗」評論（出入金、實貨交收、客服）
   - 沿用 `Broker` 實體加 `category` 維度，非新建領域（見 ADR-0045，名冊改名見 ADR-0050）

### Phase 2（市場擴張，2027+）

- 香港保險、強積金、信託公司
- 台灣、新加坡的證券商
- 全球外匯經紀商（與 WikiFX 正面競爭）

### Phase 3（多元金融，2028+）

- 加密貨幣交易所評論
- 銀行服務評論
- 機構級數據 API 訂閱

## 四、核心使用者

### 散戶投資者（C 端）

- **痛點**：不知道哪家券商穩定、不知道哪個財演真實
- **價值**：不可篡改的真實評論、KOL 鏈上勝率、爭議陪審制度
- **OpenTrade 對他們**：可信任的「投資前 due diligence 工具」

### 證券商（B 端）

- **痛點**：傳統評分平台「不付費就被攻擊」
- **價值**：透明的評論機制、有爭議可進陪審程序、不會莫名其妙被惡評
- **OpenTrade 對他們**：公平的評論場、潛在客戶觸及管道

### 財經 KOL / 財演（B 端）

- **痛點**：被指控造假無法自證、新人無法快速建立信任
- **價值**：鏈上紀錄就是最強自證，誠實 KOL 可脫穎而出
- **OpenTrade 對他們**：「可驗證的個人品牌資產」

## 五、戰略定位

### 我們不是什麼

- ❌ 不是「另一個外匯天眼」
- ❌ 不是「投資建議平台」（觸 SFC 第 4 類牌照紅線）
- ❌ 不是「交易撮合平台」（不碰 VATP 牌照）
- ❌ 不是「廣告營利平台」（不收商家錢影響評論）
- ❌ 不是「ICO / 投機代幣項目」

### 我們是什麼

- ✅ **純技術基礎設施**：區塊鏈 + IPFS 上的資訊呈現工具
- ✅ **去中心化評論協議**：平台只做 indexer，內容由用戶上鏈
- ✅ **金融服務的「OpenRice + Glassdoor」**：給 review，不給評分（避開「同牌照如何分高低」死結）
- ✅ **基於 OP Stack 的金融應用**：Base L2 為主，可擴至其他 EVM 鏈

## 六、與 WikiFX（外匯天眼）的維度差異

| 維度         | WikiFX                   | OpenTrade                                 |
| ------------ | ------------------------ | ----------------------------------------- |
| 技術基礎     | Web 2.0 中心化           | Web 3.0 去中心化（Base L2）               |
| 評論能否被刪 | 平台可刪除               | 鏈上不可刪                                |
| 評論驗證     | 平台單方面審核           | 用戶 SBT 身份驗證 + 陪審團仲裁            |
| 評分機制     | 多維打分（被質疑黑箱）   | OpenRice 式好/中/差評（透明）             |
| 商業模式     | 商家付費（被指影響公正） | B 端 SaaS 工具費 + 質押抽成（不影響評論） |
| 財演紀錄     | 無                       | 智能合約鎖定訊號歷史                      |
| 爭議處理     | 平台單方面決定           | 去中心化陪審團 + 雙方質押證據             |
| 法律風險     | 多次被指誹謗訴訟         | 平台不發表意見，只呈現鏈上資料            |

**戰略重點**：不在 Web 2.0 跟 WikiFX 拼流量（它太大），而是用「Web 3.0 不可篡改」這個它無法跟進的維度進行**降維打擊**。

## 七、商業模式（Phase 1 暫定）

> 詳見 [`02-roadmap.md`](./02-roadmap.md) 與未來的 `docs/business-model.md`

**基本原則：絕不收商家錢影響評論顯示。**

可能的收入來源（按優先序）：

1. **B 端 SaaS 工具費**：商家「認領專頁」後，可付費啟用
   - 鏈上業績儀表板
   - 客戶情緒分析
   - API 接入
2. **爭議仲裁協議費**：陪審團仲裁時雙方質押的代幣，平台抽 5-10%
3. **資料 API 授權**：累積夠多評論後，整理數據賣給保險公司、媒體、學術研究
4. **政府 / 機構合作**：協助 SFC、消委會做投訴前置篩選
5. **Phase 2+：可能引入 utility token**（需先觀察香港 VATP 監管走向）

---

## 八、合規定位

- **本身不需要 SFC 牌照**：純技術服務商，不發表意見、不撮合交易
- **避開第 4 類牌照（提供意見）**：所有 UI 標明免責聲明，不做整理推薦
- **避開 VATP 牌照**：MVP 階段使用「積分（Points）」系統，不發 token
- **PDPO 合規**：用戶資料用 zk-proof 處理，原始資料不上鏈、不長期存
- **顧問配置**：規劃中的退休 SFC 高層作為董事

---

## 九、品牌精神

- **公正（Fairness）**：技術保證，不靠承諾
- **透明（Transparency）**：所有規則、合約、資料公開可查
- **不可竄改（Immutability）**：鏈上資料任何人（包括我們自己）都改不了
- **賦權使用者（User Empowerment）**：陪審團、SBT 身份、自證喊單
