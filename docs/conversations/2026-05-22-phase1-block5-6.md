## 技術決策與注意事項

- **Brokers domain 不做 full DDD**：Phase 1 只有 read-only endpoints，presentation-only routes 直接用 Prisma。Full domain（with claim flow）待 Phase 2
- **ReviewForm 不做 chain tx**：API 端透過 outbox pattern 處理鏈上提交，前端只負責 POST to API。鏈上 transaction 由 background worker 執行（Phase 1+ 實作）
- **Console AuthGate 是 client component**：整個 layout 被 AuthGate 包住，因此 console 的 broker 頁面雖然是 Server Component，但被 client boundary 包裹。未來可考慮用 middleware 做 server-side auth redirect 提升 SSR 效率
- **API client 重複**：web 和 console 各有一份 API client，type 定義有重複。Phase 2 考慮抽到 `packages/shared` 或用 codegen

## 給未來 AI agent 的建議

- `exactOptionalPropertyTypes: true` 仍然是地雷，新增外部庫時預期 type 衝突（這次 Privy 的 `testnet?: boolean` 問題用 `as never` 繞過）
- Console 的 `AuthGate` 是純 client component，意味著所有子頁面的 Server Component 實際上在 client boundary 內運行。如果遇到 hydration 問題，這是第一個檢查點
- Brokers API 的 `_count` 型別需要 `as unknown as { _count: { reviews: number } }` 轉型，因為 Prisma 的 `include._count` 型別在 `select` 模式下不會自動推斷
