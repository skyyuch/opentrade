# ADR-0001: 採用 Base L2 作為主要區塊鏈

## Status
Accepted

## Date
2026-05-17

## Context

OpenTrade 是一個 Web 3.0 評論平台，需要把使用者評論、KOL 訊號、陪審投票等資料寫到區塊鏈以保證不可篡改。

選鏈時必須同時滿足：
1. **使用者交易成本極低**：散戶寫一條評論不能花數美金
2. **AI 開發友善**：團隊沒有專職 Web3 開發，依靠 Cursor / Claude 寫合約，需要 AI 對該鏈最熟悉
3. **EVM 相容**：Solidity + OpenZeppelin 生態最成熟
4. **HK / 政府基金認可度**：影響後續申請 CCMF / HKSTP 的基金成功率
5. **長遠可擴展**：未來可遷移 / 多鏈部署

## Decision

採用 **Base L2** 作為 OpenTrade Phase 1 的主要區塊鏈。

但**所有合約 / 前端 / 後端不寫死任何 Base 特定東西**，所有鏈相關設定走 `packages/config`，保持「OP Stack 通用」。

部署環境分配：
- **Local dev**：Anvil（Foundry 內建）
- **dev**：Base Sepolia 測試網
- **staging**：Base Sepolia 測試網
- **prod**：Base Mainnet

## Alternatives Considered

### Alternative A: Ethereum Mainnet
- **Pros**：最高的安全性與去中心化
- **Cons**：每筆 tx $5-50 USD，不可能讓散戶用
- **結論**：直接排除

### Alternative B: Arbitrum
- **Pros**：最成熟的 L2，TVL 最高，技術穩定
- **Cons**：HK 品牌認可度不如 Base（沒有 Coinbase 級的本地推廣）；費用略高
- **結論**：可考慮，但 Base 在 HK 政府基金的認可度更佳

### Alternative C: Optimism
- **Pros**：跟 Base 同 OP Stack
- **Cons**：生態小於 Base / Arbitrum，HK 認可度也不及 Base
- **結論**：未來多鏈部署的首批候選之一

### Alternative D: Polygon PoS
- **Pros**：費用極低
- **Cons**：安全模型已被質疑，正在被淘汰；不再是 L2 首選
- **結論**：不選

### Alternative E: Solana
- **Pros**：流量大，速度快
- **Cons**：使用 Rust，AI 工具鏈支援度遠不如 Solidity；EVM 不相容，未來與其他生態整合困難
- **結論**：使用者已明確排除

### Alternative F: BNB Chain
- **Pros**：費用低
- **Cons**：不同司法管轄區的監管包袱；HK / 政府基金敘事不友善
- **結論**：不選

### Alternative G: 自建 L3（基於 OP Stack）
- **Pros**：終極可控，可自定 gas token
- **Cons**：技術複雜、運維成本高、太早做；對 MVP 是 over-engineering
- **結論**：5 年後可能值得做，現在不做

## Consequences

### Positive
- Coinbase 背書帶來品牌信任，特別是申請基金 / 機構合作時
- gas 成本約 $0.0005-0.005 USD/tx，加上 Paymaster 代付後使用者體驗 = 0
- EVM 相容讓 AI 寫合約準確度極高，Solidity 教材最豐富
- OP Stack 開源，未來想多鏈或自建 L3 都容易
- 在 Base 部署的合約幾乎可零成本移植到 Optimism / 任何 OP Stack 鏈

### Negative / Trade-offs
- Base 的 sequencer 由 Coinbase 單點運行（中心化風險），但官方有 roadmap 去中心化
- 比起 Arbitrum 略晚成熟，部分 DevOps 工具（indexer 等）支援度稍弱
- Coinbase 公司風險（雖然遠低於一般 cefi 公司）

### Neutral
- 我們會用 Base Sepolia 做 dev / staging，主網部署延後到第三方 audit 之後

## Implementation Notes

- `packages/config/src/chains.ts` 必須能透過 env 切換 chainId（8453 = Base Mainnet, 84532 = Base Sepolia）
- 部署 script (`packages/contracts/script/`) 必須支援多鏈部署
- 任何前端硬編碼的「Base」字眼，必須走 i18n 或 chain config 讀取
- Privy 設定支援多 chain，初期 only allow Base，未來可加白名單
