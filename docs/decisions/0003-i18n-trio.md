# ADR-0003: i18n 三語架構：繁中（預設）/ 簡中 / 英文

## Status

Accepted

## Date

2026-05-17

## Context

OpenTrade 初期目標是香港市場，但：

- 香港有中文（繁體 + 部分簡體使用者）+ 英文使用者並存
- 中港合作項目可能引入大陸 KOL / 機構，需要簡中
- 國際投資者 / 媒體報導需要英文版
- 申請政府基金、投資人簡報需要英文 demo

如果 i18n 架構是事後補的，會造成大規模重寫。長遠正確的做法是「**從第一天就 i18n-ready，即使預設只有一種語言**」。

## Decision

從第一行 code 起，全專案採用三語 i18n 架構：

- **繁體中文（zh-Hant）** — 預設語言（HK 為主目標市場）
- **簡體中文（zh-Hans）**
- **英文（en）**

### 前端

- 使用 **next-intl**（不用 react-i18next，next-intl 對 Next.js App Router 支援最佳）
- URL 含 locale：`/zh-Hant/brokers`、`/zh-Hans/brokers`、`/en/brokers`
- 翻譯檔：`apps/web/messages/{zh-Hant,zh-Hans,en}.json`
- **絕對禁止**在程式碼裡寫死任何使用者可見的文字

### 後端

- API 錯誤訊息回傳 `error_code`（如 `REVIEW_NOT_FOUND`），由前端 i18n 翻譯
- 業務 metadata（券商名稱介紹等）DB 多語言欄位設計：
  ```ts
  name: {
    zh_hant: string;
    zh_hans: string;
    en: string;
  }
  ```

### 用戶生成內容（評論）

- 儲存原文 + 偵測來源語言
- SQS + Lambda 背景任務呼叫 AI 翻譯成另兩種語言
- 翻譯結果存 `review_translations` 表（cached）
- 前端按使用者 locale 顯示翻譯版，**標明「機器翻譯」**並提供切換到原文按鈕

### 鏈上資料

- 只存原文 hash，**翻譯不上鏈**（因為 AI 翻譯結果可能變化）

### 預設與備援

- 預設語言：zh-Hant
- 翻譯不全時的備援順序：zh-Hant → en → zh-Hans
- 從 `Accept-Language` header 自動偵測

## Alternatives Considered

### Alternative A: 只做繁中，未來再加

- **Pros**：MVP 速度最快
- **Cons**：未來補做要重寫所有頁面與後端錯誤處理；違反「徹底長遠方案」原則
- **結論**：不選

### Alternative B: 用 react-i18next 而非 next-intl

- **Pros**：生態最大
- **Cons**：對 Next.js 14 App Router + Server Components 支援不如 next-intl；URL routing 整合較複雜
- **結論**：不選

### Alternative C: 加上日文 / 韓文 / 西文等更多語言

- **Pros**：更國際
- **Cons**：MVP 階段沒有對應市場，純粹增加維護成本
- **結論**：保留架構彈性，但暫不加

### Alternative D: 把翻譯存到鏈上

- **Pros**：去中心化
- **Cons**：AI 翻譯會變化，鏈上存翻譯會造成資料不一致；且翻譯本身不需要不可篡改性（原文才需要）
- **結論**：不選

## Consequences

### Positive

- 從第一天就 i18n-ready，不用回頭重寫
- URL 含 locale 對 SEO 多語言極友善
- DB 多語言欄位設計支援彈性
- 鏈上只存原文 hash，避免翻譯漂移問題
- 將來新增第 4、5 種語言成本極低（只需加翻譯檔）

### Negative / Trade-offs

- 前期開發每個 string 都要寫進 messages 檔，比直接寫死慢一點
- DB schema 比單語言複雜（多欄位）
- 增加 AI 翻譯成本（DeepL / OpenAI）
- 內容審核變複雜：審原文還是審翻譯？

### Neutral

- 三語都要做品牌名 / slogan 設計，需要文案資源

## Implementation Notes

### 文化敏感事項

- 香港使用「繁體中文」但**用詞與台灣不完全一樣**（例如：質素 vs 品質、信用卡 vs 信用咭）
  - 建議由 HK 在地團隊成員 review 翻譯
- 簡中市場需注意敏感字詞（金融類用詞需符合中國大陸規範）
- 英文用 American English 為基準

### 預設行為

- 第一次造訪：根據 Accept-Language → zh-Hant > zh-Hans > en
- 使用者可手動切換並記憶到 cookie
- Cookie 名：`ot-locale`，1 年過期

### 嚴禁事項

- ❌ 在 React 元件裡 `<div>提交評論</div>` 直接寫死
- ❌ 在 API 裡 `throw new Error('找不到評論')` 寫死中文
- ❌ DB 任何 user-facing 欄位只有單語版
- ❌ 將 AI 翻譯後的內容上鏈

### 翻譯服務優先序

1. **DeepL**（主）：質量最好，付費 API
2. **OpenAI GPT-4**（備援）：偶爾質量更好，但較貴
3. **Google Translate API**（最後備援）
