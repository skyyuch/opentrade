## 一、目前已完成（讓股東看到進度）

- **鏈上評論（產品核心）**：散戶可登入 → 找券商 → 寫評論 → 內容上鏈不可改。完成。
- **KOL 訊號上鏈**：KOL 可發買/賣/持有訊號上鏈，系統自動算勝率。完成。
- **金商 / 貴金屬交易商**：沿用券商架構，香港金商名冊已建檔。完成。
- **KOL 類別**：KOL 可依「類型（財演 / 技術指標賣家）」+「領域（股票 / 加密 / 外匯）」兩軸分類，用戶可在 KOL 目錄篩選。完成（ADR-0053，功能 + 完整測試金字塔：unit + component + e2e）。
- **投訴（前半）**：用戶可提交投訴 + 證據，後台可核實/駁回。完成。
- **內容審核（第一層）**：發佈前自動擋粗口/人身攻擊/引流廣告/違法/起底，且負評用語一律放行（不刪評論）。完成。
- **商戶後台基礎 + 公開回應**：商戶可認領專頁、回應評論。完成。

## 二、還沒開發的項目 + 建議次序 + 預計時間

### 1. 券商比較 — 中型，約 1 週

- 讓用戶並排比較 2-3 家券商的客觀資料：牌照、好/中/差評分佈、已核實投訴數。
- 幾乎只用現有資料，畫面豐富，最貼合「投資前盡職調查工具」定位，demo 加分。
- 紅線：只比事實、不做總分排名、不接受商戶付費影響排序。

### 2. 陪審團 + 投訴仲裁（Phase 3，核心差異化）— 大型，約 4-6 週

- 補上投訴的「仲裁迴圈」：被投訴方答辯 → 鏈上隨機抽 5 位陪審員 → 投票 → 結果上鏈不可改。
- 這是對 WikiFX 降維打擊、跟股東/評審講故事最強的一張牌（去中心化仲裁，平台無權干預）。
- 工程最大（智能合約 + 鏈上隨機 + 陪審員身份 + 前後端），建議拆多階段推進。
- 前提：需要先備好「創始陪審員」名單（業界前輩），屬於人脈/招募工作，可與開發並行。
- 對應 [`02-roadmap.md`](./02-roadmap.md) 的 **Phase 3：MVP-C 陪審團 V1**。

### 3. 報價 — 小-中，先做 1 份合規評估（ADR）+ 約 1 週

- 建議做法：**不自建即時行情**（數據授權貴 + 有觸監管疑慮），改為外連/嵌入第三方「延遲行情」並標明「非投資建議」。
- 動工前先寫一份合規邊界文件讓大家拍板，再做輕量呈現。
- 契合度較低，排在核心功能之後。

### 4. 新聞 — 小型，先做 1 份合規評估（ADR）+ 約 1 週

- 建議做法：只做**中立的第三方新聞標題聚合 / 外連**，不轉載全文（版權）、不挑選排序（避免被當成投資建議）。
- 與「不可篡改評論」核心最遠，策略價值最低，排最後。

## 四、整體時程概覽

- KOL 類別已完成（✅）。剩餘功能線若**依序**做：券商比較 → 陪審團/投訴仲裁 → 報價 → 新聞，約 **8-11 週**。
- 陪審團是時程主導項（4-6 週），建議排在創科局 Q3 申請窗口（7/1–9/30）前完成到可 demo。
